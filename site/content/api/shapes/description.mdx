# Shapes
Shapes are geographical lines. Shapes are used by <Link href="/reference/trips">trips</Link> to represent the physical path a bus takes.

There are two different formats for shapes:

1. ### GTFS Shape Object
    <Link href="https://gtfs.org/documentation/schedule/reference/#shapestxt" target="_blank" rel="noopener noreferrer">The GTFS definition of a shape.</Link> This shape contains an array of shape points. Each shape point has a sequence, a latitude and longitude point, distance traveled, and stopId.

2. ### Encoded Polyline
    A method of encoding a series of coordinates into a string via a lossy compression algorithm created by Google. An encoded polyline looks like this:

		`ezsFv{qyO@fDbK?CfI`

	See <a href="https://developers.google.com/maps/documentation/utilities/polylinealgorithm" target="_blank" rel="noopener noreferrer">Google's Encoded Polyline Algorithm documentation</a> for more information on how to encode and decode these strings, as well as the <a href="https://developers.google.com/maps/documentation/utilities/polylineutility" target="_blank" rel="noopener noreferrer">Interactive Polyline Utility</a>.
